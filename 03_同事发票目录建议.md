# 同事发票目录建议 Draft

这份说明给同事看，用来统一“发票到底放哪里”。

## 推荐放置位置

我建议每位同事在自己电脑上建立独立目录：

`~/Documents/报销管理/hongda_reimbursement_runtime`

待处理材料放入：

`~/Documents/报销管理/hongda_reimbursement_runtime/inbox`

每个项目单独建文件夹：

`inbox/YYYYMM_项目名称_项目编号/`

示例：

`inbox/202606_某项目_HDC202606001/`

项目文件夹里建议这样分：

```text
202606_某项目_HDC202606001/
  发票/
  附件/
  行程单/
  说明/
```

## 为什么这样放

1. 不容易丢  
   Documents 目录比 Downloads、Desktop 和聊天缓存更适合长期保存。

2. 权限更稳定  
   WorkBuddy 更容易读取 Documents 下的固定目录，也更容易写入 output。

3. 减少混票  
   每个项目独立文件夹，能减少不同客户、不同月份、不同项目混在一起的风险。

4. 方便人工复核  
   报销结果都在同一个 runtime 的 output 里，人工打开最新 run 目录即可检查。

5. 方便备份和迁移  
   只要备份整个 `报销管理` 目录，就能保留原始材料、配置和历史 Draft 输出。

6. 方便后续自动化  
   文件夹命名规范后，脚本更容易识别报销月份、项目名称、项目编号和材料分类。

## 不推荐的位置

- `Downloads/`：下载文件太杂，容易被清理。
- `Desktop/`：桌面临时文件多，容易误拖误删。
- 微信、企业微信、邮箱附件缓存：路径不稳定，文件可能过期或被清理。
- 多人共用同一个 `inbox/`：容易混票，责任不清。

## 如果同事已经有自己的发票文件夹

可以保留原文件夹作为“原始收集区”，但建议复制一份到 runtime 的 `inbox/项目文件夹/` 做处理。这样原始收集区不受脚本影响，也方便形成统一报销工作区。

## 命名建议

项目文件夹：

`YYYYMM_项目名称_项目编号`

材料文件：

可以保留原始文件名，不强制改名。若人工整理，建议包含日期、费用类型、金额，例如：

`2026-06-18_打车费_35.00.pdf`

但注意：不要为了自动化而改动已归档的原始发票。WorkBuddy 默认只读，不会要求同事重命名原件。
